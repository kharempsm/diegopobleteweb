Overcoming Inconsistent Tag Triggering in a Single-Page Application Environment

Associated with BNP Paribas Cardif / Jul 2022 - Jan 2023

I recently worked on a project involving several websites built as Single-Page Applications (SPAs). SPAs present unique challenges for web analytics tracking because users can navigate between different views without triggering traditional page load events. This can lead to inaccurate data on page views and user interactions, making it difficult to understand user behavior and optimize the website effectively.


My main objective was to ensure accurate tracking of user navigation and interaction within these SPAs. I also needed to implement a robust system for triggering tags in specific locations, even without traditional page loads.


To address these challenges, I took the following steps:


  • Leveraging History Change Triggers: I utilized GTM's "History Change" trigger to capture changes in the URL fragment identifier. This allowed me to track user navigation between different views within the SPA, even though full page reloads weren't occurring.

  • Implementing Data Layer: I worked closely with the development team to implement a comprehensive data layer that captured key user interactions, such as button clicks, form submissions, and video plays.

  • Custom Event Tracking: I configured custom events within GTM to capture specific user interactions and trigger tags based on these events. This enabled precise control over tag firing, ensuring that tags were activated only when specific conditions were met.

  • Thorough Testing and Validation: I rigorously tested the implementation, using debugging to verify that tags were firing correctly and data was being captured accurately.

And this resulted in:


  • Accurate user journey tracking: We gained a clear understanding of how users navigate through the SPA, identifying popular content and potential areas for improvement.

  • Enhanced tag management: We achieved granular control over tag firing, ensuring that tags were activated only when necessary and that data was collected efficiently.

Ensuring Accurate Data Collection by Correcting GTM and Data Layer Implementation

Associated with Google / Jan 2020 - Mar 2020

While working on a website optimization project some time ago, I encountered an issue that hindered accurate data collection. The development team had incorrectly implemented the GTM container snippet and data layer, which led to inconsistent and unreliable data in our analytics reports.


The problem stemmed from the placement of the data layer variable after the GTM container snippet in the website's code. This meant that GTM was trying to access data from the data layer before it was actually available.


To address this, I had to explain the issue clearly and guide them on the correct implementation.


Explanation:


I started by emphasizing the importance of the data layer as a communication bridge between the website and GTM. I explained that GTM relies on the data layer to access information about user interactions and page events.


Then, I highlighted the specific issue: the data layer variable was being declared after the GTM container snippet. This meant that when GTM tried to access data from the data layer, it wasn't available yet, leading to errors and inaccurate tracking.


Finally, I provided clear instructions on the correct implementation:


  • Declare the data layer variable before the GTM container snippet. This ensures that the data layer is available when GTM needs it.

  • Use dataLayer.push() to add dynamic data. For information that becomes available after the page loads (e.g., link clicks), use dataLayer.push() to add it to the data layer without overwriting existing data.

After this clarification and guidance, the developers corrected the implementation. This resulted in:


  • Accurate data collection: GTM could now access the data layer correctly, leading to accurate and reliable data in our analytics reports.

  • Improved data integrity: We had confidence that the data we were collecting accurately reflected user behavior on the website.

  • Enhanced decision-making: With accurate data, we could make informed decisions about website optimization.
